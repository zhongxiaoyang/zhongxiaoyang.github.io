公众平台对接系列01 - 授权获取用户信息

发表于 2018-11-30 | 分类于 JavaScript ， NodeJs

这段时间在流量项目做后台，流量项目需要快速迭代小功能小实验，不断尝试新玩法。对技术要求没那么高。这段时间的很多功能需要公众号作为载体，用了很多的是授权获取用户信息做其他事情，也算是闲置无聊，记下来吧

前期准备

  • 经过ICP备案的域名
  • 经过认证的服务号
  • 属于自己的线上服务，使用测试号测试的时候可以本地跑的服务，使用域名生成工具生成一个临时域名指向该服务。

测试环境资源

  • 测试公众号地址

https://mp.weixin.qq.com/debug/cgi-bin/sandbox?t=sandbox/login

微信扫码登录之后就拥有一个属于自己的测试号了，测试号拥有与订阅号的几乎所有功能以及服务号的部分功能，因为是测试环境，域名无需ICP备案就可以使用

  • 测试域名

要去买个域名难免都会有些麻烦，测试的时候可以不用做这个麻烦的工作，当然如果你已经有域名了，甚至域名已经备案了，那事情就相当容易了，如果没有域名，我们可以使用工具临时生成一个随机域名，这个域名指向自己当前IP，并且外网可以访问。

工具地址：

https://github.com/localtunnel/localtunnel

需要注意的是，端口4001是我们后端项目占用的端口，说白了就是我们本地应该使用localhost:4001访问的地址已经可以通过https://popular-mule-93.localtunnel.me来访问，并且外网可以访问。但是这个方式生成的临时域名肯定是未备案的，这个毋庸置疑，其次是这个域名很快就会失效，并没有具体研究失效情况，快的话10分钟，慢的话你可以使用1、2个小时。

如果失效了，就再次执行命令lt --port 4001生成新的域名

公众号配置

基本配置

开发 -> 基本配置 可以配置 公众号开发信息 (包括重置AppSecret、设置IP白名单)、服务器配置

开发者ID(AppID)和开发者密码(AppSecret)

开发者需要牢记开发者ID(AppID)和开发者密码(AppSecret)，后期开发经常会用到。测试号的开发者密码(AppSecret)不隐藏，非测试号“为保障帐号安全，公众平台不再储存AppSecret；如果遗忘，请重置”

配置IP白名单

为了提高公众平台开发者接口调用的安全性，避免一旦开发者ID和密码泄露后给帐号造成损失。我们对调用“获取access_token”接口增加IP白名单校验：只有将IP地址设置为公众号的IP白名单，才能成功调用该接口。测试号不需要设置IP白名单

参考：https://mp.weixin.qq.com/cgi-bin/announce?action=getannouncement&key=1495617578&version=1&lang=zh_CN&platform=2

未设置白名单会报错：40164, invalid ip

服务器配置

可以参考：https://mp.weixin.qq.com/wiki?t=resource/res_main&id=mp1421135319，连接已经讲的很详细了。

验证消息是否来自微信服务器主要有两个流程：

  • 将token、timestamp、nonce三个参数进行字典序排序
  • 将三个参数字符串拼接成一个字符串进行sha1加密
  • 开发者获得加密后的字符串可与signature对比，标识该请求来源于微信
  • 若确认此次GET请求来自微信服务器，请原样返回echostr参数内容，则接入生效，成为开发者成功，否则接入失败

我们首先调用checkRequest方法，该方法很简单：

  • 将token(服务器配置填写的token)、接口请求参数中的timestamp和nonce参数三个参数组成一个数据，按照字典序排序后拼接成字符串
  • 将拼接后的字符串使用sha1加密(这里用到了sha1库)
  • 将加密后的结果字符串与signature参数作比较，相同则表示消息来源于微信服务器
  • 最终结果返回echostr参数原文，表示api已确认消息确实来源于微信服务器。所以当你发现消息不是来源于微信服务器而且你觉得应该拒绝认证，那你就不能返回echostr参数原文了。反过来说，如果你不care消息是否来源于微信服务器，啥都不想做又想通过认证，该api只需简单的吧echostr参数返回调用方即可。

JS安全域名

安全域名配置过程很简单，进入公众号设置的功能设置里填写域名，这里可以填写3个安全域名，一个月可以做三个安全域名修改，所以说一个公众号一个月最多可以使用9个安全域名。

配置安全域名需要下载一个TXT文件(一个包括一串字符串的普通文本文件)，并且让安全域名根路径下可以访问该文件。假如TXT文件名称为MP_verify_aaa.txt，域名为https://abc.domain.com，这里配置的时候需要填写为abc.domain.com，并且可以通过https://abc.domain.com/MP_verify_aaa.txt访问该文件。我暂时的做法是把文件放到项目静态文件目录，比如node的静态文件设置的是public文件，就放public下。

为啥需要设置安全域名？

图片上说了，开发者可在域名下调用微信开发的JS接口。

微信JS-SDK是微信公众平台面向网页开发者提供的基于微信内的网页开发工具包。通过使用微信JS-SDK，网页开发者可借助微信高效地使用拍照、选图、语音、位置等手机系统的能力，同时可以直接使用微信分享、扫一扫、卡券、支付等微信特有的能力，为微信用户提供更优质的网页体验。

但是这个功能不是没有门槛用户可以无节操随便使用的(比如诱导分享是不允许的)，微信基于安全考虑，需要配置JSSDK安全域名，只要你页面中用到了JSSDK。

发现诱导分享行为时，微信可以根据此域名追溯到所有分享出去的链接，以及通过这些链接增加的粉丝。这样，微信就可以牢牢控制了你的微信平台，一旦发现违规，让分享链接失效，删除掉诱导行为增加的粉丝，是瞬间就可以完成的(也就是地址被封)。因此，微信平台的开发者，一定要合理来使用分享功能，不要因小失大。等到你的微信平台被封，估计哭都来不及了

可以参考微信JS-SDK文档：https://mp.weixin.qq.com/wiki?t=resource/res_main&id=mp1421141115

业务域名

配置了业务域名可以防止页面输入的时候弹出安全警告的文案影响用户体验，域名配置个数个次数与JS安全域名一致。也是需要下载一个TXT文件并放在域名根目录下。

网页授权域名

这是服务号有的功能，服务号可以使用网页授权获取用户微信信息(微信昵称、性别、国家省市信息、头像地址等信息)，订阅号没有该功能。

配置的域名可以访问TXT文件。这里只能配置一个域名，配置的域名要跟获取code之后跳转的redirect_uri的域名一致

本地使用测试号的时候可以简单使用本地局域网IP + 服务端口号即可，或者使用临时域名生成工具生成的临时域名。

注意事项

业务域名、JS安全域名已经网页授权域名一般都是填写前端服务器域名。但是建议前端老域名在 nginx 做重定向到新域名，这样前后端都配置一样的域名

业务域名和JS安全域名都可以配置3个，这个就可以一个后端服务同时提供给多个业务使用。但是授权域名只需配置一个，多个业务应该公用一个授权结果。

网页授权获取用户信息

当我们最初在公众号上操作的时候（点击菜单跳转h5），经常会弹出一个授权提示的微信页面，类似下图

这就是网页授权获取用户信息功能，我们个公众号要获取【用户昵称、头像、地区和性别】信息的时候需要用户手动授权，授权流程包括

第一步：用户同意授权，获取code

第二步：通过code换取网页授权access_token

第三步：刷新access_token（如果需要）

第四步：拉取用户信息(需scope为 snsapi_userinfo)

附：检验授权凭证（access_token）是否有效

不需要用户手动授权的网页授权

网页授权有两种类型

  • 只获取用户的openid，不需要用户手动授权

也就是说用户是无感知的，对应的scope参数为【snsapi_base】

  • 另一种除了获取openid，还获取用户更多的信息，这种网页授权需要用户手动授权。

对应的scope参数为【snsapi_userinfo】

获取code

前端页面【https://frontend.com/page/auth】访问连接：

https://open.weixin.qq.com/connect/oauth2/authorize?

​ appid=wxxxxxxxxxx53&

​ redirect_uri=https://frontend.com/page/auth&

​ response_type=code&

​ scope=【snsapi_base || snsapi_userinfo】

注意redirect_uri参数的域名也网页授权域名配置的域名必须是一致的。前端访问这一地址，如果scope为snsapi_userinfo就会弹出授权页面给用户授权，授权之后会跳转页面带上code参数【https://frontend.com/page/auth&code=xxxxxxxx】，注意code参数我们需要用到的。

我不会前端，本地测试的时候为了能顺利拿到code，我在网页授权接口域名配置里配置了后台域名，并且这里的redirect_uri参数也用了后台服务的一个地址，所以我直接看后台的log就可以拿到code了，【测试号网页授权域名配置可以配置本地局域网IP+端口】

注意code只能用一次，不能重复使用

根据code获取access_token

从打印出来的日志看出，无论scope取那个值，获取access_token返回的数据都是一样的，即包括【openid和access_token】具体他们的差别是获取其他信息

拉取用户信息

继上一步得到的access_token，我们可以使用它进一步获取用户的其他信息，前提要求授权方式为【snsapi_userinfo】

我们细细看【co-wechat-oauth】包和【co-wechat-api】包的api，会发现他们都有获取用户信息的方法【getUser】，但是返回的结果不一样，甚至数据也不完全一致。至于关注公众号后授权和关注公众号前授权，数据是一样的，微信对授权不强求用户必须关注了公众号才能授权

测试号需要关注了才能测试授权获取用户信息，对于未关注公众号又强制授权会出现什么情况只能用真实的服务号来测试。下面是代码执行后测试号和服务号各个情况打印的结果

订阅号获取用户信息

熟悉订阅号的人应该清楚订阅号是不能接开发的，我们是无法使用网页授权获取用户信息的，但是我们可以通过其他方法同样得到用户信息。

这里需要讲下网页授权获取用户信息有什么好处，网页授权获取用户信息是在我们的服务器端触发的，也就是在我们h5页面中触发的，既然是在我们的页面，我们就可以知道这个用户是谁，也就是用户的userId，这样我们就可以将userId绑定微信的用户信息。

那么订阅号怎么获取用户信息呢，这个需要用户主动与我们公众号交互，比如关注，发消息等，这些交互都会形成一个事件，我们可以通过这些事件进一步得到用户信息
