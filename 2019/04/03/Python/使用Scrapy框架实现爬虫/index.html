Spider

细看前面编写的demo_spider.py，短短的几行代码就可以爬虫一个网页，这也算是最简单的一个Spider了。每个Spider就是一个python类，定义一个Spider，我们需要继承 scrapy.Spider类，定义一些属性和覆写一些方法。

  • name属性

用于区别Spider。 该名字必须是唯一的，您不可以为不同的Spider设定相同的名字。

  • start_urls

Spider启动时爬取的url列表，每个url会创建一个scrapy.http.Request对象，并且以parse()方法作为回调函数，回调调用parse()方法的时候会传递

  • parse()

是spider的一个方法。 被调用时，每个初始URL完成下载后生成的Response对象将会作为唯一的参数传递给该函数。 该方法负责解析返回的数据(response data)，提取数据(生成item)以及生成需要进一步处理的URL的 Request 对象

Spider运行过程

我们查看Spider类的源码，会发现Spider类必须设置name属性，如果没有设置，运行会报错，另外当没有显示设置start_urls参数的时候默认为空数组

其次，Spider提供了一些生命周期方法，如果我们需要在响应的生命周期方法中执行一些代码，这样覆写该方法即可

当我们运行启动命令的时候，scrapy会读取name执行的Spider，找到Spider后读取start_urls的url列表并逐个执行下面下载（如果url列表为空，则parse方法不会被调用，很容易理解，没有网页需要爬取）。

  • start_requests方法

这里会先执行start_requests方法来获取request，它是从start_urls中读取url。当指定了url的时候，改方法会调用make_requests_from_url方法来创建Request对象

  • make_requests_from_url方法

该方法接受一个url返回用于爬取的Request对象，该方法仅仅会被scrapy调用一次，可以覆写它把它改造成生成器，执行一些初始化的操作。默认未被覆写的情况下，该方法返回的Request对象中， parse方法作为回调函数，dont_filter参数也被设置为开启

  • parse方法

当response没有指定回调函数时，该方法是Scrapy处理下载的response的默认方法。

parse负责处理response并返回处理的数据以及(/或)跟进的URL。 Spider对其他的Request的回调函数也有相同的要求。

该方法及其他的Request回调函数必须返回一个包含 Request、dict 或Item的可迭代的对象。

  • closed方法

当spider关闭时，该函数被调用。 该方法提供了一个替代调用signals.connect()来监听 spider_closed信号的快捷方式

  • log方法

日志打印，log中自带Spider的name属性

scrapy.http.Response对象

最终我们从response对象获取目标数据，response对象提供了很多属性和方法供我们使用。

  • response.url - 请求的网页地址
  • response.headers - 响应的头信息
  • response.status - 响应状态码，默认200
  • response.body - 响应体，返回的是一个非Unicode的字符串
  • response.request - 原Request对象，可以使用对属性追溯原请求信息

内容选取 - Selectors

爬虫最常见的是从页面中提取数据，现有的一些库可以达到这个目的：

  • BeautifulSoup 是在程序员间非常流行的网页分析库，它基于HTML代码的结构来构造一个Python对象， 对不良标记的处理也非常合理，但它有一个缺点：慢。
  • lxml 是一个基于 ElementTree (不是Python标准库的一部分)的python化的XML解析库(也可以解析HTML)。

Scrapy提取数据有自己的一套机制。它们被称作选择器(seletors)，因为他们通过特定的 XPath 或者 CSS 表达式来“选择” HTML文件中的某个部分。

XPath 是一门用来在XML文件中选择节点的语言，也可以用在HTML上。 CSS 是一门将HTML文档样式化的语言。选择器由它定义，并与特定的HTML元素的样式相关连。

Scrapy选择器构建于 lxml 库之上，这意味着它们在速度和解析准确性上非常相似。

保存数据

我们能够正常的爬取网页了，也知道了从网页中获取数据的方法，现在我们需要把获取到的数据保存起来。为了定义常用的输出数据，Scrapy提供了 Item 类。 Item 对象是种简单的容器，保存了爬取到得数据

许多Scrapy组件使用了Item提供的额外信息: exporter根据Item声明的字段来导出数据、 序列化可以通过Item字段的元数据(metadata)来定义、 trackref 追踪Item实例来帮助寻找内存泄露 (see 使用 trackref 调试内存泄露) 等等。
